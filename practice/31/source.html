Users who own their backend

The architecture of the web originally focused on allowing independent publishers to have their own web site on which they publish public ontent. In the web 2.0 architecture, bigger web sites started hosting user content - public as well as limited-audience and private. In the meantime, desktop and mobile operating systems did allow users to own their data storage - at least the local data storage on a hard disk or memory card.

What we're trying to do with unhosted web apps and per-user backends, is to change the web to a software platform on which owning your data is possible. Unhosted web apps by themselves allow you to use the local on-device data storage, but to add device-to-device sync and cloud backup, it is necessary for the user to own some place where this data can be stored: a per-user backend.

Per-user storage

The first and most important requirement for per-user backend is storage. There are a number of providers of proprietary per-user cloud storage: Google's GoogleDrive, Microsoft's OneDrive, Apple's iCloud, Dropbox Dropbox Platform. Of these, Google and Microsoft offer cloud storage as an addition to their offering of cloud services; Microsoft and Apple offer cloud storage sign up from the operating systems that are pre-installed on Windows and iOS/OSX devices respectively, and Dropbox is the only one in this list who offer proprietary per-user cloud storage.

Google and Dropbox offer cross-origin APIs, which are accessible from unhosted web apps like export.5apps.com.
